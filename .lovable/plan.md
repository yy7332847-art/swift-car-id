# نظام تشييك اللوحات بالصوت (Plate Voice Check)

تطبيق ويب بواجهة موبايل متجاوبة 100% (شريط تنقل سفلي، تصميم عصري) لمحصّلي البنوك: يرفعون ملف Excel يومي بلوحات السيارات، ثم يفتحون "المسجّل" فيقولون أرقام اللوحات صوتياً، والنظام يستخرج اللوحات فوراً ويطابقها مع القاعدة ويعرض النتائج مع تقارير PDF/Excel.

## المراحل

### 1) البنية الأساسية
- تفعيل Lovable Cloud (قاعدة بيانات + مصادقة + تخزين).
- نظام مصادقة كامل: تسجيل، دخول، خروج، إعادة تعيين كلمة السر.
- جدول `profiles` + جدول `user_roles` (admin / user) — الأدوار بجدول منفصل (منع تصعيد الصلاحيات).
- جدول `subscriptions` (تاريخ بداية، مدة بالأيام، مفعّل/معطّل).
- سياسات RLS كاملة على كل الجداول.

### 2) لوحة الأدمن
- حساب أدمن تجريبي جاهز (سأعطيك email/password).
- صفحة يرى فيها كل المستخدمين، حالة الاشتراك، عدد الأيام المتبقية.
- أزرار: **تفعيل اشتراك ٢٠ يوم**، **تفعيل مخصّص**، **تعطيل الحساب**، **إعادة تفعيل**.
- المستخدم المعطّل/المنتهي اشتراكه يرى صفحة "الحساب معطّل — تواصل مع الإدارة".

### 3) رفع ملف Excel والبيانات
- شاشة رفع Excel (الأعمدة: اللوحة، البنك، النوع، الهيكل، التاريخ).
- تحليل الملف على الخادم وحفظ اللوحات في جدول `plates` مرتبط بالمستخدم مع رقم دفعة/تاريخ.
- إمكانية عرض النسخ السابقة (تاريخ الرفع) والرجوع لأي نسخة.
- تطبيع رقم اللوحة (إزالة مسافات، توحيد أحرف عربية شكلاً) للبحث السريع.

### 4) شاشة التسجيل الصوتي (القلب)
- شاشة كبيرة بزر ميكروفون واحد، عدّاد لحظي، وقائمة اللوحات المكتشفة.
- تسجيل صوتي مستمر عبر Web Audio API → WAV 16kHz mono (متوافق مع Safari/Chrome).
- Streaming Speech-to-Text عبر Lovable AI Gateway (`openai/gpt-4o-transcribe` مع `stream:true`) — لغة عربية، مع prompt موجّه: "استخرج فقط أرقام لوحات سيارات سعودية: ٣ حروف + ٤ أرقام. تجاهل أي كلام آخر".
- بارسر ذكي على كل delta:
  - يحوّل الأرقام المنطوقة (اثنين وأربعين ألفاً ومائتين واثنين وعشرين → 42222) — يدعم اللهجات المصرية/السعودية.
  - يقبل قراءة الأرقام يسار→يمين أو العكس.
  - يستخرج نمط: 3 حروف عربية + 4 أرقام.
  - يطابق فوراً مع القاعدة → لو موجود يعرض ✓ أخضر مع تفاصيل السيارة، لو ناقص حرف/رقم يسجّله كـ "غير مكتمل" مع الوقت.
- كل لوحة مكتشفة تظهر كبطاقة قابلة للنقر → مواصفات كاملة (النوع، البنك، رقم الهيكل).

### 5) الجلسة والتقارير
- كل جلسة تسجيل تُحفظ (start, end, total_spoken, matched_count).
- بعد انتهاء الجلسة: ملخّص (مثلاً "١٠٠٠ لوحة، ٥ مطابقات، ٣ غير مكتملة") + قائمة كل اللوحات المكتشفة مع الوقت.
- تصدير تقرير PDF و Excel للجلسة.
- سجل تاريخي لكل الجلسات السابقة.

### 6) الواجهة (موبايل أولاً)
- شريط تنقل سفلي: **الرئيسية**، **رفع ملف**، **تسجيل** (زر وسط بارز)، **الجلسات**، **حسابي**.
- ألوان عصرية (أزرق داكن + أخضر نيون للمطابقة + أحمر للنقص)، أنيميشن Motion سلس، Haptic-like feedback بصري عند كل مطابقة.
- كل الشاشات مقيّدة عرضاً بحجم موبايل حتى على الديسكتوب (max-width mobile frame).

## التفاصيل التقنية
- Stack: TanStack Start + React 19 + Tailwind v4 + shadcn.
- Backend: Lovable Cloud (Supabase) — جداول: `profiles`, `user_roles`, `subscriptions`, `plate_batches`, `plates`, `recognition_sessions`, `detected_plates`.
- AI: Lovable AI Gateway — `openai/gpt-4o-transcribe` (streaming, عربي).
- PDF: `jspdf` + `jspdf-autotable`، Excel: `xlsx`.
- Audio: Web Audio API + AudioWorklet لتسجيل WAV 16kHz mono موثوق على iOS/Android.
- الحماية: RLS على كل جدول، فحص الاشتراك في middleware على الخادم، الأدمن يُتحقّق منه عبر `has_role()` security definer.

## ما سأسلّمه بعد التنفيذ
- رابط التطبيق شغّال.
- بيانات دخول الأدمن (email + password).
- شرح مختصر للتدفق: أدمن يفعّل مستخدم → مستخدم يرفع Excel → يفتح المسجّل → يتكلم → يشوف المطابقات → يصدّر تقرير.
