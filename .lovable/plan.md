# خطة إصلاح PWA من الألف للياء

## الهدف
تحويل تجربة التثبيت إلى PWA حقيقي ومستقر: زر التثبيت يفتح نافذة المتصفح الأصلية عندما يكون المتصفح مؤهلاً، مع دعم iOS/Android/Desktop، وكاش ذكي، وتحديثات Service Worker بدون شاشة بيضاء.

## ما سيتم إصلاحه
1. إصلاح سبب شاشة الخطأ البيضاء: خطأ React #418 سببه عدم تطابق التحميل بين HTML المرسل والمتصفح، وسأمنع شاشة الخطأ من الظهور بسبب أخطاء hydration القابلة للاسترداد.
2. تحسين زر التثبيت: التقاط حدث `beforeinstallprompt` مبكراً قبل تحميل React حتى لا يضيع، ثم استخدامه عند الضغط على زر التثبيت.
3. توضيح الحالات: إذا المتصفح لم يرسل إذن التثبيت بعد، الزر لا يقول “غير مدعوم” خطأً؛ بل يوضح حالة التثبيت أو يعطي خطوات النظام الصحيح.
4. تطبيق PWA كامل: manifest صحيح، أيقونات كاملة، تسجيل Service Worker محمي من بيئة المعاينة، وتحديث سلس بعد موافقة المستخدم.
5. Offline caching: كاش ذكي للصفحات الأساسية والـ assets، مع عدم كاش API أو بيانات حساسة، ورسالة واضحة عند انقطاع الاتصال.
6. اختبار التوافق بالكود: فحص manifest، الأيقونات، تسجيل SW، ومسارات assets النسبية بعد التعديل.

## ملاحظات مهمة للمستخدم
- التثبيت الحقيقي يعمل على الرابط المنشور HTTPS في تاب مستقل، وليس داخل iframe المعاينة داخل محرر Lovable.
- المتصفح وحده هو الذي يسمح بظهور نافذة التثبيت الأصلية؛ التطبيق لا يستطيع إجبارها بدون حدث `beforeinstallprompt` من المتصفح.
- على iPhone لا يوجد prompt تلقائي مثل Chrome؛ iOS يتطلب Share ثم Add to Home Screen حسب قيود Apple.

## النتيجة المتوقعة
عند فتح الرابط المنشور HTTPS والضغط على زر التثبيت بعد جاهزية المتصفح، تظهر نافذة التثبيت الأصلية. وعند وجود تحديث جديد، تظهر رسالة تحديث داخل التطبيق، وبعد موافقتك يتم التحديث بسلاسة.
