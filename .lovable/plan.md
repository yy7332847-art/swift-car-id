# خطة إنشاء PWA حقيقي ومستقر

## 1) إصلاح التثبيت الحقيقي
- التقاط حدث التثبيت `beforeinstallprompt` مبكراً قبل تحميل React حتى لا يضيع الحدث.
- زر التثبيت يستخدم نافذة المتصفح الأصلية عند توفرها، وليس رسالة “غير مدعوم” بشكل خاطئ.
- على iPhone/iPad، عرض خطوات iOS الرسمية لأن Safari لا يفتح نافذة تثبيت تلقائية مثل Chrome/Edge.
- توضيح حالة المعاينة: التثبيت يعمل من رابط HTTPS منشور ومفتوح في تبويب مستقل؛ لا يحتاج دومين خاص، ورابط Lovable المنشور يكفي. المعاينة داخل iframe قد تمنع نافذة التثبيت.

## 2) Manifest وأيقونات التطبيق
- استخدام `manifest.webmanifest` باسم التطبيق العربي، اتجاه RTL، وألوان التطبيق.
- ربط كل أحجام الأيقونات المطلوبة: 192/256/384/512، maskable، Apple touch icon، favicon.
- الحفاظ على `display: standalone` ليظهر كتطبيق مستقل عند التثبيت.

## 3) Service Worker آمن ومولّد
- استبدال Service Worker اليدوي بتوليد قياسي عبر `vite-plugin-pwa`.
- منع التسجيل في التطوير ومعاينة Lovable وداخل iframe وداخل Capacitor، حتى لا تتكرر مشاكل الكاش القديم أو الشاشة البيضاء.
- دعم kill-switch عبر `?sw=off` لإلغاء عامل الخدمة عند الحاجة.

## 4) تحديثات التطبيق بدون كسر المستخدم
- عند توفر إصدار جديد، تظهر رسالة عربية واضحة: “تحديث جديد متاح”.
- لا يتم فرض التحديث فجأة؛ المستخدم يضغط “تحديث الآن”.
- بعد الموافقة، يتم تفعيل النسخة الجديدة وإعادة تحميل التطبيق بسلاسة.

## 5) Offline Mode و caching ذكي
- الصفحات الأساسية والتنقلات تعمل باستراتيجية NetworkFirst حتى يحصل المستخدم على الأحدث عند وجود إنترنت.
- ملفات البناء الثابتة تستخدم CacheFirst لأنها ملفات hashed وآمنة للكاش.
- الأيقونات والـ manifest تستخدم StaleWhileRevalidate.
- عند جاهزية التشغيل بدون إنترنت تظهر رسالة داخل التطبيق.
- مؤشر الاتصال الموجود داخل التطبيق يستمر في عرض حالة عدم توفر الاتصال والمزامنة.

## 6) منع تكرار شاشة الخطأ البيضاء
- إزالة التقاط أخطاء `window.error` و `unhandledrejection` من الـ Native error boundary لأنها كانت تحوّل أخطاء hydration العابرة مثل React #418 إلى شاشة توقف.
- بقاء Error Boundary فقط لأخطاء React الحقيقية داخل الشجرة.
- الحفاظ على صفحة خطأ لطيفة في الحالات الحقيقية مع زر متابعة وإعادة تحميل.

## 7) اختبارات القبول
- TypeScript clean بدون أخطاء.
- التأكد أن `public/sw.js` اليدوي أزيل حتى لا يتعارض مع Service Worker المولّد.
- التأكد أن زر التثبيت لا يقرأ `window` أثناء أول render لتجنب hydration mismatch.
- عند النشر على HTTPS وفتح الرابط في تبويب مستقل، نافذة التثبيت الأصلية تظهر عندما يعتبر المتصفح التطبيق installable.

## ملاحظة مهمة
لا يوجد أي متصفح يسمح للموقع بتثبيت التطبيق قسرياً بدون موافقة المستخدم. الصحيح هو: المستخدم يضغط الزر، المتصفح يفتح نافذة التثبيت الأصلية، ثم المستخدم يوافق. رابط Lovable المنشور HTTPS يكفي؛ لا يلزم شراء دومين خاص.
